## §0 규약

- 컨텍스트: ContextA / ContextB / ... (도메인 영역 분할)
- 페르소나: 정책서 §페르소나 그대로 사용. 한글 → 영문 mapping이 필요한 경우 본 섹션에서 명시
  (예: 시청자(회원) → `시청자회원`, 시청자(비회원) → `시청자비회원`)
- 화면 ID: `<컨텍스트>_<페르소나>_NN` (페르소나별 NN은 01부터 2자리 패딩)
- 액션 ID: `<페르소나>:Action:N` (페르소나별 1부터 증가) — §1 통합 표 "사용자 액션" 열에 정의하고, 다이어그램 엣지 라벨에도 등장시킨다
- 색상 범례:
  - 빨강(error): 에러/실패/충돌
  - 노랑(unclear): 정책서 미정의 — 회의 결정 필요
  - 보라(auth): 인증/권한 게이트
  - 점선(system): 시스템 트리거

## §1 페르소나별 흐름

> 각 페르소나 = ① 흐름 요약 + ② 다이어그램(주역) + ③ 화면·액션 통합 표.
> 화면 전환 관계와 엣지 케이스는 별도 표 없이 다이어그램이 표현한다.

### <페르소나> — <컨텍스트>

flowchart TD

    %% 사용자 액션은 엣지 라벨에 "<액션ID> <단계명>" 형식으로 표기
